## Production Deployment Notes

### Before Deploying to Railway:

1. **Run database migration:**
   ```bash
   npm run db:migrate
   ```
   This applies Workstream J timestamps & indexes.

2. **Set environment variables:**
   - `ADMIN_PASSWORD` - for production admin auth
   - `SESSION_SECRET` - for session cookies
   - `LOG_LEVEL=info` - for structured logging
   - `CACHE_TTL` - optional, defaults to 300 (5 minutes)

3. **Verify:**
   - `/health` endpoint returns 200
   - Admin login works in production
   - Image optimization active (`/images/test.png?w=400`)
