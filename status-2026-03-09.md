# Status: Multi-Repo Journal Generation — 2026-03-09

## What We Built

A local-only workflow that fetches commits from all public GitHub repos under `blaine-2050` and generates weekly dev-journal blog posts, with duplicate tracking.

**Run it:** `npm run journal:generate`

## Things to Think About

1. **All posts are drafts.** Generated posts have `draft: true` in frontmatter. You'll want to review and publish selectively — some weeks may have trivial commits not worth a post.

2. **Post quality varies.** The generator uses conventional commit classification (feat/fix/docs/etc). Repos without conventional commits get everything under "Other". Consider editing posts before publishing, or adding a minimum commit threshold.

3. **Private repos are excluded.** The GitHub API call only fetches public repos. If you want to include private repos, the `gh` CLI already has auth — just change the API query parameter from `type=public` to `type=all`.

4. **Local-only projects (not on GitHub).** The existing `git.ts` can read any local git repo. To process local-only projects, you'd need to either: (a) push them to GitHub, or (b) add a local repo scanner to the multi-repo script. Option (b) is ~20 lines of code.

5. **Naming convention.** New posts use `dev-journal-<repo>-week-of-YYYY-MM-DD.md`. The two original hand-written posts use `dev-journal-YYYY-MM-DD-<suffix>.md`. These coexist safely — the tracking system uses week dates, not filenames.

6. **Reprocessing a week.** If you want to regenerate a post (e.g., after editing the generator template), manually remove the week from the repo's tracking JSON and re-run.

7. **No new dependencies.** Uses `gh` CLI (already authenticated) and Node built-ins only.
